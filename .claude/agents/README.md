# Agents

복잡하고 다단계 작업을 처리하는 전문 에이전트입니다.

---

## Agents란?

Agents는 특정 복잡한 작업을 처리하는 자율적인 Claude 인스턴스입니다. 스킬(인라인 가이드 제공)과 달리 에이전트는:

- 별도의 하위 작업으로 실행됩니다
- 최소한의 감독으로 자율적으로 작동합니다
- 전문 도구에 접근할 수 있습니다
- 완료 시 포괄적인 보고서를 반환합니다

**주요 장점:** 에이전트는 **독립적**입니다 - `.md` 파일만 복사하면 즉시 사용할 수 있습니다!

---

## 사용 가능한 에이전트

### frontend-error-fixer

**목적:** 프론트엔드 에러 진단 및 수정

**사용 시기:**

- 빌드 과정에서 발생하는 에러 (TypeScript, bundling, linting 에러)
- 브라우저 콘솔에서 발생하는 런타임 에러 (JavaScript 에러, React 에러)
- 네트워크 및 API 통합 문제
- CSS/스타일링 충돌 및 렌더링 문제

**특징:**

- 빌드 타임 및 런타임 에러 모두 처리
- TypeScript/JavaScript 에러 진단 및 해결
- React 19 에러 바운더리 및 일반적인 함정 처리
- Biome linter 및 formatter 문제 해결
- Next.js 15, Turbopack 빌드 도구 문제 해결

**통합:** ✅ 그대로 복사하여 사용

## 에이전트 통합 방법

### 표준 통합 (대부분의 에이전트)

**1단계: 파일 복사**

```bash
cp showcase/.claude/agents/agent-name.md \
   .claude/agents/
```

**2단계: 확인 (선택사항)**

```bash
# 하드코딩된 경로 확인
grep -n "~/git/\|/root/git/\|/Users/" .claude/agents/agent-name.md
```

**3단계: 사용**

Claude에게 요청: "[agent-name] 에이전트를 사용하여 [작업] 수행"

끝입니다! 에이전트는 즉시 작동합니다.

---

### 커스터마이징이 필요한 에이전트

일부 에이전트는 프로젝트에 맞게 커스터마이징이 필요할 수 있습니다:

- 하드코딩된 경로가 있는 경우 업데이트 필요
- 특정 인증 설정이 필요한 경우 프로젝트에 맞게 조정
- 스크린샷 경로 등 프로젝트별 설정이 필요한 경우 수정

에이전트 파일을 확인하고 필요에 따라 수정하세요.

## 에이전트 vs 스킬 사용 시기

| 에이전트 사용 시...      | 스킬 사용 시...      |
| ------------------------ | -------------------- |
| 작업에 여러 단계 필요    | 인라인 가이드 필요   |
| 복잡한 분석 필요         | 모범 사례 확인       |
| 자율적 작업 선호         | 제어 유지 원함       |
| 작업에 명확한 목표 있음  | 지속적인 개발 작업   |
| 예: "모든 컨트롤러 검토" | 예: "새 라우트 생성" |

**둘 다 함께 사용 가능:**

- 스킬은 개발 중 패턴 제공
- 에이전트는 완료 시 결과 검토

## Claude Code를 위해

**사용자를 위해 에이전트를 통합할 때:**

1. **[AGENTS.md](../../AGENTS.md) 읽기**

2. **`.md` 파일만 복사** - 에이전트는 독립적입니다

3. **하드코딩된 경로 확인:**

   ```bash
   grep "~/git/\|/root/" .claude/agents/agent-name.md
   ```

4. **경로가 발견되면** `$CLAUDE_PROJECT_DIR` 또는 `.`로 업데이트

5. **인증 에이전트의 경우:** 먼저 JWT 쿠키 인증을 사용하는지 확인

**끝입니다!** 에이전트는 통합하기 가장 쉬운 컴포넌트입니다.

## 나만의 에이전트 만들기

에이전트는 선택적 YAML frontmatter가 있는 마크다운 파일입니다:

```markdown
# 에이전트 이름

## 목적

이 에이전트가 하는 일

## 지침

자율 실행을 위한 단계별 지침

## 사용 가능한 도구

이 에이전트가 사용할 수 있는 도구 목록

## 예상 출력

결과를 반환할 형식
```

**팁:**

- 지침을 매우 구체적으로 작성
- 복잡한 작업을 번호가 매겨진 단계로 분해
- 정확히 무엇을 반환할지 명시
- 좋은 출력의 예시 포함
- 사용 가능한 도구를 명시적으로 나열

## 문제 해결

### 에이전트를 찾을 수 없음

**확인:**

```bash
# 에이전트 파일이 있는가?
ls -la .claude/agents/[agent-name].md
```

### 에이전트가 경로 에러로 실패

**하드코딩된 경로 확인:**

```bash
grep "~/\|/root/\|/Users/" .claude/agents/[agent-name].md
```

**수정:**

```bash
sed -i 's|~/git/.*project|$CLAUDE_PROJECT_DIR|g' .claude/agents/[agent-name].md
```
